# 周易卦师｜数字起卦

**Zhouyi Digital Diviner**

输入一个具体问题和三组数字，自动生成本卦、动爻与变卦，  
并结合《周易》卦辞、爻辞，提供克制、清晰、可落地的决策参考。

> 三数起卦，观势明心；不作断言，只助判断。

## 项目简介

`zhouyi-digital-diviner` 是一个面向 AI Agent、Codex、Skill 平台及其他支持自定义技能平台的开源中文 Skill。

用户只需提供：

1. 一个明确、具体的问题；
2. 三组凭第一直觉想到的整数；

Skill 即可生成：

- 本卦；
- 动爻；
- 变卦；
- 本卦卦辞；
- 对应动爻爻辞；
- 卦象与现实问题的综合解读；
- 风险隐患、行动时机与现实验证点；
- 2—4 条可执行建议。

本项目不把卦象描述为确定预言，也不鼓励用户依赖占卜代替现实判断。

## 核心特点

### 固定的三数组卦规则

- 第一组数字：计算下卦；
- 第二组数字：计算上卦；
- 第三组数字：计算动爻；
- 翻转动爻阴阳后生成变卦。

### 起卦脚本辅助计算

通过 `scripts/cast_hexagram.py` 计算：

- 上卦与下卦；
- 本卦名称；
- 动爻位置；
- 动爻阴阳变化；
- 变卦名称。

这样可以降低模型心算时出现错卦、错爻和错变卦的概率。

### 强制引用经典原文

完整解读时必须引用：

- 本卦卦辞；
- 本次动爻对应的爻辞；
- 卦辞简明解读；
- 爻辞简明解读。

原文优先从本地资料库读取，不允许用模糊记忆拼接或编造。

### 关注现实判断

每次完整解读尽量包含：

- 当前状态；
- 有利条件；
- 风险隐患；
- 行动时机；
- 现实验证点；
- 可执行建议；
- 有倾向但保留条件的结论。

## 快速体验

向 Skill 发送一个明确问题和三组数字：

```text
我近期是否适合换工作？73、5227、1
```

也可以使用空格、逗号、顿号或换行分隔：

```text
这个合作项目目前是否适合继续推进？

35
128
6
```

Skill 会自动输出：

```text
本卦
动爻
变卦
卦辞原文与解读
爻辞原文与解读
卦象分析
具体问题解读
建议
结论
```

## 推荐提问方式

问题越具体，解读越容易落到现实判断。

### 推荐

```text
我是否适合在未来三个月内更换工作？
```

```text
这个合作项目目前是否值得继续投入？
```

```text
我和对方的关系，接下来应主动沟通还是暂时观察？
```

```text
这个新业务方向，目前适合直接扩大投入，还是先做小范围验证？
```

### 不推荐

```text
我的一生怎么样？
```

```text
我以后会不会成功？
```

```text
我什么时候发财？
```

建议一次只询问一件具体事项。

## 安装方法

### 上传完整 Skill 压缩包

适用于支持自定义 Skill 上传的平台。

1. 下载或克隆完整项目；
2. 保持原有目录结构；
3. 将项目文件夹压缩为 ZIP；
4. 在对应平台上传完整 Skill 包；
5. 不要只上传 `SKILL.md`。

必须确保以下文件同时存在：

```text
SKILL.md
scripts/cast_hexagram.py
references/interpretation-guide.md
references/zhouyi-original-text.md
```

只上传 `SKILL.md` 可能导致：

- 起卦脚本无法调用；
- 卦辞、爻辞原文库缺失；
- 输出重新依赖模型记忆；
- 漏引或错引经典原文。

### 放入本地 Skill 工作区

示意结构：

```text
.agents/
└─ skills/
   └─ zhouyi-digital-diviner/
      ├─ SKILL.md
      ├─ LICENSE
      ├─ scripts/
      └─ references/
```

不同工具的 Skill 目录位置可能不同，请以对应平台的实际规则为准。

### 手动测试脚本

确保本地已安装 Python 3，在项目根目录运行：

```bash
python scripts/cast_hexagram.py 73 5227 1
```

若脚本能正常返回本卦、动爻和变卦信息，说明基础计算功能可用。

## 起卦规则

### 八卦数字对应

| 数字 | 卦 | 象 |
|---:|---|---|
| 1 | 乾 | 天 |
| 2 | 兑 | 泽 |
| 3 | 离 | 火 |
| 4 | 震 | 雷 |
| 5 | 巽 | 风 |
| 6 | 坎 | 水 |
| 7 | 艮 | 山 |
| 8 | 坤 | 地 |

### 下卦与上卦

第一组数字除以 8 取余，计算下卦；第二组数字使用同样规则计算上卦。

- 余数为 1—7时，直接对应八卦；
- 余数为 0时，按 8 计算，对应坤卦。

### 动爻

第三组数字除以 6 取余：

- 余数为 1—5时，对应初爻至五爻；
- 余数为 0时，按 6 计算，对应上爻。

### 变卦

翻转动爻阴阳：

- 阳爻变阴爻；
- 阴爻变阳爻。

重新识别上下卦后，得到变卦。

## 输出结构

完整解读默认包含：

```markdown
# 六爻占卜解读

**本卦：卦名（上卦上、下卦下）**

## 经典原文

### 本卦卦辞
> 卦辞原文

**卦辞解读：**
……

### 动爻爻辞
> 对应爻辞原文

**爻辞解读：**
……

## 卦象分析

**动爻：规范爻位名称（阴变阳／阳变阴）**

**变卦：卦名（上卦上、下卦下）**

## 具体问题解读

## 建议

## 结论

曰：□□□□，□□□□。

卦象仅为决策辅助，建议结合现实情况综合判断。
```

结尾“曰”句严格使用两个四字短句，例如：

```text
曰：慎始守正，循序成器。
```

## 深入追问与重新起卦

完成一次解读后，如继续追问同一事项的更深层问题，需要重新提供三组数字。

例如：

```text
那我接下来具体应该先做什么？32、717、5
```

Skill 会先独立解读新卦，再结合此前卦象综合分析。

询问另一件事情时，发送：

```text
重新起卦
```

## 方法边界

本项目使用“三数组卦法”，用于生成本卦、一个动爻和变卦。

它不是完整的传统纳甲六爻排盘工具，不包含：

- 纳甲；
- 六亲；
- 世应；
- 月建；
- 日辰；
- 用神；
- 伏神；
- 旺衰生克；
- 神煞体系。

它也不声称完整复现传统梅花易数中的：

- 体卦与用卦；
- 互卦；
- 时间起卦；
- 方位起卦；
- 五行旺衰；
- 完整体用生克体系。

本项目更准确的定位是：

> 以三组数字生成本卦、动爻与变卦，并结合《周易》卦辞爻辞进行现实问题梳理的辅助工具。

## 常见问题

### 为什么不是“完整六爻排盘”？

因为本项目不计算纳甲、六亲、世应、日月建和用神，因此不应冒充完整传统六爻排盘。

### 为什么必须保留原文库？

语言模型可能漏引、错引或混淆相邻爻辞。本地原文库用于提高经典原文引用的稳定性和可核验性。

### 为什么不直接告诉用户“吉”或“凶”？

《周易》中的吉凶通常包含条件、阶段和行为要求。单纯输出“能或不能”，容易忽略现实环境和用户可以采取的行动。

### 可以连续问很多次同一件事吗？

不建议短时间内反复起卦追问同一个问题。更适合在问题发生实质变化，或需要询问新的具体层面时重新起卦。

### 可以直接输入卦名吗？

可以。例如：

```text
山风蛊，三爻动
```

如果只提供本卦、没有动爻，Skill 只解读本卦，不虚构动爻和变卦。

### 原文库是学术校勘本吗？

不是。本项目中的经文资料用于 Skill 引用和基础核验，采用常见通行文本整理，不替代专业古籍整理、版本校勘和学术研究。

## 已知限制

- AI 对现实问题的理解依赖用户提供的信息；
- 即使卦象计算正确，现实解读仍可能存在偏差；
- 不同《周易》传本之间可能存在少量异文；
- 部分平台可能限制 Python 脚本或本地资料读取；
- 不同模型对 Skill 规则的遵循程度可能不同；
- 本项目不提供完整传统六爻或梅花易数专业排盘能力。

欢迎通过 Issue 提交：

- 错卦；
- 错爻；
- 错变卦；
- 原文错字；
- 爻辞对应错误；
- 输出结构缺失；
- 其他可复现的问题。
